# How to AI Coding

AIコーディングツールの実用ガイド。

## おすすめツール

| ツール | タイプ | 入手方法 |
|--------|--------|----------|
| [Google Antigravity](https://antigravity.google/) | IDE + Agent | 直接ダウンロード、個人無料 |
| [GitHub Copilot](https://github.com/features/copilot) | IDE拡張 | 学生無料、一般は有料 |
| [Continue](https://continue.dev/) | IDE拡張 | オープンソース無料 |

初心者はAntigravityから始めるのがおすすめ。API設定不要ですぐ使える。

---

## 使い方

### Step 1: AGENTS.mdをコピーする

このリポジトリの [AGENTS.md](./AGENTS.md) を開いて、内容をすべてコピーする。

### Step 2: 自分のプロジェクトに貼り付ける

自分のプロジェクトのルートディレクトリに `AGENTS.md` ファイルを作成し、コピーした内容を貼り付ける。

```
your-project/
├── AGENTS.md    <-- ここに貼り付け
├── src/
├── package.json
└── ...
```

### Step 3: 必要に応じてカスタマイズ

プロジェクトに合わせて、AGENTS.mdの内容を調整する。例えば：
- 使用言語の指定
- 特定のファイルやディレクトリへの制限
- プロジェクト固有のルール追加

### Step 4: AI codingを開始

ツールを起動してプロジェクトを開く。AIはAGENTS.mdを読み込み、ルールに従って動作する。

## AGENTS.mdとは

AGENTS.mdはAI agentへの指示書である。このファイルをプロジェクトに置くことで、AIの動作をコントロールできる。

主な内容：
- AIの役割と仕事範囲
- ファイル修正のルール
- コーディング前に提案を行う形式
- パッケージ管理のルール
- テストのルール

詳細は [AGENTS.md](./AGENTS.md) を参照。

---

## DEVLOG.md（開発ログ）とは

開発の記録を残すファイル。AIが作業の区切りごとに自動的に記録を追記する。

記録される内容：
- 実施した作業の内容
- 変更したファイルのリスト
- 次回への引き継ぎ事項

これにより、過去に何を開発したかを振り返ることができる。チームでの引き継ぎにも役立つ。

---

## WHITEPAPER.md（技術白書）とは

各機能の技術的な実装詳細を記録するファイル。AIが新機能を実装した際に自動的に記録を追記する。

記録される内容：
- 機能の概要と目的
- 技術的なアプローチ
- 使用した技術やライブラリ
- アーキテクチャとデータフロー

これにより、後から「この機能はどう実装されているか」を理解しやすくなる。新しいメンバーのオンボーディングにも役立つ。

---

## 注意点

- AIが生成したコードを鵜呑みにしない。必ずレビューする
- パスワードやAPI keyなどの機密情報をAIに送らない
- 一度に大量のファイルを変更させない。問題発生時のデバッグが困難になる
- gitを使う。いつでもロールバックできるように
